Techtonic Design System

A brand vision existed as ten beautiful PDFs. I built the system that ships it — 670 components, three platforms, zero drift.

Lead Product Designer Design Systems Flutter, React Native, Web Token Architecture
Rock. Solid. Dollars.

A complete brand vision — strategy, identity, motion, UI principles — distilled into three words. Beautiful as intent. Missing as code.

The Gap

PDFs don’t ship products.

We’d partnered with Ragged Edge — the London agency behind the Wise rebrand — on brand strategy and identity. Ten PDFs covering everything from verbal identity to motion design.

The brand described how ZAR should feel — heavy, grounded, certain. How the UI should behave — anchored grids, stone textures, no glass effects. How motion should land — with the weight of rock, not the bounce of rubber.

All of this existed as intent. None of it existed as code. My job was to close that gap.

The Brand Vision

A geological metaphor for money.

The brand isn’t a fintech identity. It’s an idea: money should feel solid. Like rock. Like something you can hold. This materializes as four UI principles:

01
Structure
Strict tectonic grid. Elements feel anchored. Nothing floats or drifts.
02
Physicality
No glassy blurs or transparency. Solid and tangible. No lightweight aesthetics.
03
Rocks
Stone textures on dollar bills and balances. Dollars look like they weigh something.
04
Haptics
Roomy tap targets. Strong physical feedback. Not bouncy. Solid.

The Migration

Killing common_widgets.

Building a new system is half the work. Killing the old one is the other half. The inherited codebase had no documentation, no naming convention — just hex values scattered across files.

Strategy: parallel run, then cut. New features used Techtonic exclusively. Existing screens migrated one at a time. When a component hit zero references — delete.

The AI Multiplier

Design systems are AI infrastructure.

When I built the Engineer Self-Serve workflow, the AI needed a design system to work with. Not descriptions — the actual system. Components with real names. Tokens with real values.

Techtonic was that system. 670 components accessible via MCP. When Chothu (our AI design consultant) recommends a screen, it uses specific Techtonic components by name.

Learnings

Four lessons.

01
Vision without infrastructure is a PDF on a shelf.
The best measure of a brand is whether the 50th screen an engineer builds looks as intentional as the first.
02
Naming is governance.
Tanzanite/Core instead of Color(0xFF1A73E8). The name carries intent and makes drift visible in every PR.
03
Kill the old system or it kills the new one.
As long as the legacy existed, engineers had a choice. Decommission day made adoption universal overnight.
04
Design systems are AI infrastructure.
Built for consistency. Turned out to be the reason AI-assisted design worked. Good systems are inherently machine-readable.
